Tu vas faire une passe finale de finition sur la landing page mobile-first de METABO AI.

Important :
La landing est déjà bien meilleure qu’avant.
Je ne veux pas une refonte complète.
Je veux une passe de polish haut de gamme, très ciblée, pour corriger les derniers défauts visibles et faire passer la page d’un bon niveau à un niveau premium fini.

Objectif :
Améliorer les détails de hiérarchie, densité, lisibilité, perception premium et conversion mobile, sans casser la structure actuelle.

Contexte produit :
METABO AI est une application iPhone qui transforme les données Apple Santé en lecture claire et utile.
Positionnement :
premium, sobre, santé, rassurant, élégant, mobile-first, style Apple-like.

Ce qu’il faut corriger maintenant :

1. Sélecteur de langue
Actuellement, le “EN” seul dans le header mobile n’est pas assez clair.
Il donne une impression incomplète ou ambiguë.

À faire :
- remplacer le “EN” seul par une solution plus claire et plus premium
- options acceptées :
  - un switch discret “FR / EN”
  - ou un sélecteur langue dans le menu burger
  - ou une icône globe discrète avec menu de langue
- sur mobile, éviter tout élément ambigu dans le header
- la solution doit être élégante, compacte et cohérente avec le style du site

2. Hero : remonter légèrement la preuve visuelle
Le hero est bon, mais le mockup iPhone pourrait apparaître plus tôt dans le premier écran.

À faire :
- réduire légèrement l’espace entre le CTA secondaire et le mockup
- remonter le mockup iPhone dans la hero
- optimiser la hauteur verticale de la hero sur mobile
- garder une respiration premium, sans tasser brutalement le contenu
- l’utilisateur doit percevoir plus vite qu’il s’agit d’une vraie app

3. Section “Simple. Automatique. Lisible.”
Cette section est encore un peu trop étalée verticalement.
Les gros chiffres 1, 2, 3 créent trop de vide sur mobile.

À faire :
- compacter cette section
- réduire les espacements verticaux
- réduire l’impact visuel excessif des grands chiffres si nécessaire
- améliorer le rythme de lecture sur mobile
- conserver l’idée des 3 étapes, mais avec une densité plus maîtrisée
- cette section doit être rapide à lire, pas longue à traverser

4. Section METABO+ : alléger le scroll mobile
La section premium est bonne, mais elle reste encore un peu lourde en hauteur sur mobile.

À faire :
- compacter les cartes tarifaires
- réduire les hauteurs excessives
- mieux équilibrer les marges internes
- garder la sensation premium
- rendre la lecture plus efficace

Concernant le comparatif :
- le tableau est lisible mais trop long verticalement
- rendre cette partie plus compacte et plus intelligente sur mobile
- possibilités :
  - limiter l’affichage mobile aux points les plus importants
  - ajouter un bouton “Voir le comparatif complet”
  - transformer une partie en accordéon ou en version condensée
- objectif : réduire la fatigue de scroll tout en gardant la clarté de l’offre

5. Footer : renforcer légèrement la lisibilité
Le footer est propre, mais certains textes gris sur fond sombre manquent un peu de contraste.

À faire :
- augmenter légèrement le contraste des textes secondaires
- garder l’esthétique premium sombre
- améliorer la lisibilité sans casser l’équilibre visuel

6. Finition générale
Faire une vraie passe de polish sur :
- alignements
- hiérarchie typographique
- cohérence des espacements
- densité mobile
- lisibilité immédiate
- perception “produit fini”

Ce que je veux préserver absolument :
- la structure actuelle globale
- le style premium sobre
- le header mobile propre
- le hero actuel
- le mockup iPhone
- les cartes bénéfices
- la section confidentialité sombre
- le footer élégant

Ce que je ne veux pas :
- pas de refonte totale
- pas d’effets visuels tape-à-l’œil
- pas d’animations inutiles
- pas de surcharge marketing
- pas de design startup générique
- pas de régression responsive

Priorité UX :
Le site doit sembler conçu pour un iPhone moderne, avec une sensation de finition haut de gamme.
Chaque écran doit paraître net, maîtrisé, léger, crédible.

Travail attendu :
1. analyse rapide des points à ajuster
2. modification ciblée du code existant
3. amélioration visible mais subtile
4. résumé final clair des ajustements effectués

Checklist finale à valider avant de terminer :
- le header mobile est plus clair, surtout sur la langue
- le mockup apparaît plus tôt dans la hero
- la section “Simple. Automatique. Lisible.” est plus compacte
- la section METABO+ est moins lourde sur mobile
- le comparatif est plus intelligent à parcourir
- le footer est plus lisible
- l’ensemble paraît plus premium et plus fini qu’avant
